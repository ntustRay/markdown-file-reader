Introduction to the Project

Published
• Last updated 2 days ago • 5 min read

Welcome to the documentation for Project Nebula. This library aims to provide a robust set of tools for building scalable web applications with minimal configuration. It is designed to be intuitive for beginners while powerful enough for enterprise needs.

info

Important Note

This documentation covers version 2.0.1. If you are migrating from v1.x, please check the Migration Guide first.

# Quick Start

To get started with Project Nebula, you'll need to install the package via npm or yarn. Ensure you have Node.js version 14 or higher installed on your machine.

bash 
# Install via npm
npm install @project-nebula/core

# Or via yarn
yarn add @project-nebula/core

# Core Features

  • check_circle Zero Configuration: Works out of the box with sensible defaults for most modern web stacks.
  • check_circle Type Safe: Built completely in TypeScript, providing you with first-class type definitions.
  • check_circle Modular Architecture: Import only what you need. Tree-shaking is supported by default.

# Configuration

You can create a `nebula.config.js` file in the root of your project to customize the behavior.

nebula.config.js 
module.exports = {
  theme: 'dark',
  plugins: ['@nebula/plugin-auth'],
  experimental: true
}
